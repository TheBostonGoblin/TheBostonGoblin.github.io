RIT X NASA Gamification and Performance-Based Assessment(GPBA)

Description

A multi-year project the GPBA has been in development since 2020 The main goal of the project was to assist astronauts in training their sensorimotor skills and retraining astronauts in regaining these skills after prolonged periods in space. This project was a joint venture between, Computer Engineers,Electrical Engineers, Biomedical Engineers, and Game Development Students. As engineers worked on hardware to test astronauts ability to maintain balance, game developer created assessments, in the form of games to test sensorimotor and balance skills. We would go down to Houston,TX to give a portable version of the hardware to Mr.Scott, as well as, give a presentation of our work, and receive a special tour from Mr.Scott himself! Agreeing to send over the software after its completion at the end of the semester. We worked hard to get the project ready not only for the Houston Trip but for RIT's IMAGINE, an event that showcased our hard work as well as the work of thousands of other students. This project has been by far the most exciting and enjoyable. I only wish I could have worked on it more.

Game Engine Used

Unity 2020.3.14f

Role

I was a Game Developer & Designer.
After being interviewed and being reviecving the privalge to work on as a part time member of the team. I was tasked with revising the river level which in the past had been used as a test bed and therefore was polluted with empty scripts,unused assets, uncommented code, and overly complicated and overscoped code. I would clean these aspects up and get the level ready to be shipped and showcased to the Johnson Space Center and IMAGINE RIT. The Game was developed in Unity/C#.

Lab Central Career Diagram

Description

I was tasked with creating a interactive and dynamic career diagram for the LabCentral ignite team. This was to be implimented into their already pre-existing squarespace page. The Diagram purpose is to show basic information on roles within the LabCentral, as well as show future roles that can be obtained given thier current position.

Library(s) Used

  • D3.JS

Role

I was a Front-End Web Developer For Labcentral.
The first thing I did was to find a suitable library to help me create the diagram as utilizing canvas would take too long and lack the necessary functionality I need to implement the diagram properly. D3.JS was the best library I found to complete this goal as not only could it be used to create complicated diagrams and graphics, but it could also be used to make dynamic and interactive diagrams. It also helped that the library is open source and free to use, given this information I first began experimenting and learning how D3.JS worked, watching YouTube videos, looking at code snippets, and reading D3.JS documentation. I Finally put the information to use creating a simple diagram. information on positions would be stored in a JSON file. Interactivity and changes to the diagram were handled by javascript and D3.JS. In the end, the diagram was functional, but the code and how information was stored in JSON was redundant and unnecessarily long. In the future, I would like to come back to this project and refine it. NOTE:(The current version of this project has placeholder information and not all roles are accurate) I also worked on data analysis with my boss, reported progress on the webpage, and towards the end of my internship met with web developers who would be taking over the project once I left. These developers primarily gave feedback on my work thus far and offered advice on the current design of the diagram.
